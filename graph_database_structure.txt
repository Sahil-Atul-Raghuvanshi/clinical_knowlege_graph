OVERVIEW
================================================================================

This document describes the complete structure of the clinical knowledge graph,
including all node types (labels), their properties, and the relationships
between them. The graph represents patient journeys through hospital systems,
capturing admissions, transfers, clinical events, diagnoses, procedures,
medications, lab results, and more.


================================================================================
NODE LABELS AND DESCRIPTIONS
================================================================================

1. PATIENT
   Description: Core patient demographic information
   Properties:
     - subject_id (unique identifier)
     - name = "Patient"
     - gender
     - anchor_age
     - anchor_year
     - anchor_year_group
     - dod (date of death, if applicable)
     - total_number_of_admissions

2. HOSPITAL ADMISSION
   Description: Represents a hospital admission episode
   Properties:
     - hadm_id (unique identifier)
     - subject_id
     - name = "HospitalAdmission"
     - admittime
     - dischtime
     - deathtime
     - admission_type
     - admit_provider_id
     - admission_location
     - discharge_location
     - insurance
     - language
     - marital_status
     - race
     - edregtime
     - edouttime
     - hospital_expire_flag
     - hospital_admission_sequence_number

3. EMERGENCY DEPARTMENT
   Description: Emergency department visit
   Properties:
     - event_id (stay_id)
     - subject_id
     - hadm_id (if linked to admission)
     - name = "Emergency Department"
     - intime
     - outtime
     - period (human-readable duration)
     - disposition
     - arrival_transport

4. UNIT ADMISSION
   Description: Admission to a specific hospital unit
   Properties:
     - event_id (transfer_id)
     - subject_id
     - hadm_id
     - transfer_id
     - name = "UnitAdmission"
     - eventtype = "admit"
     - careunit
     - intime
     - outtime
     - period (human-readable duration)
     - service_given

5. ICU STAY
   Description: Intensive Care Unit stay (dual label with UnitAdmission)
   Labels: :UnitAdmission:ICUStay
   Properties:
     - (inherits all UnitAdmission properties)
     - first_careunit
     - last_careunit
     - los (length of stay in days)

6. TRANSFER
   Description: Patient transfer between units
   Properties:
     - event_id (transfer_id)
     - subject_id
     - hadm_id
     - transfer_id
     - name = "Transfer"
     - eventtype = "transfer"
     - careunit
     - intime
     - outtime
     - period
     - service_given

7. DISCHARGE
   Description: Patient discharge event
   Properties:
     - event_id (transfer_id)
     - hadm_id
     - subject_id
     - transfer_id
     - name = "Discharge"
     - eventtype = "discharge"
     - careunit
     - intime
     - outtime
     - period
     - service_given

8. PRESCRIPTIONS BATCH
   Description: Container node for all prescriptions during an event
   Properties:
     - event_id
     - name = "PrescriptionsBatch"

9. PRESCRIPTION
   Description: Group of medicines prescribed at the same time
   Properties:
     - event_id
     - starttime
     - name = "Prescription"
     - medicines (array of formatted medicine strings)
     - medicine_count

10. PREVIOUS PRESCRIPTION MEDS
    Description: Medications patient was taking before ED visit
    Properties:
      - stay_id
      - name = "PreviousPrescriptionMeds"
      - medications (array of "Medication Name - Classification")
      - medication_count
      - charttime

11. ADMINISTERED MEDS
    Description: Medications administered during ED visit
    Properties:
      - stay_id
      - name = "AdministeredMeds"
      - medications (array of medication names)
      - medication_count
      - charttime

12. PROCEDURES BATCH
    Description: Container node for all procedures during an event
    Properties:
      - event_id
      - hadm_id
      - subject_id
      - name = "Procedures"

13. PROCEDURES
    Description: Group of procedures performed at the same time
    Properties:
      - event_id
      - time
      - name = "Procedures"
      - procedures (array of formatted procedure strings)
      - procedure_count
      - source ("ICU_EVENTS" or "ICD")

14. DIAGNOSIS
    Description: Patient diagnoses
    Properties:
      - event_id
      - hadm_id
      - subject_id
      - name = "Diagnosis"
      - complete_diagnosis (array of diagnosis titles)
      - diagnosis_count
      - ed_diagnosis (true for ED diagnoses)
      - primary_diagnoses (array)
      - secondary_diagnoses (array)
      - primary_count
      - secondary_count

15. LAB EVENTS
    Description: Container node for all lab events during an event
    Properties:
      - event_id
      - hadm_id
      - subject_id
      - name = "LabEvents"

16. LAB EVENT
    Description: Group of lab tests performed at the same time
    Properties:
      - event_id
      - hadm_id
      - subject_id
      - charttime
      - name = "LabEvent"
      - lab_results (array of formatted lab result strings)
      - lab_count

17. DRG (Diagnosis Related Group)
    Description: DRG code assigned to hospital admission
    Properties:
      - drg_id (unique: type_code)
      - name = "DRG"
      - drg_type
      - drg_code
      - description
      - drg_severity
      - drg_mortality

18. MICROBIOLOGY EVENTS
    Description: Container node for all microbiology events during an event
    Properties:
      - event_id
      - hadm_id
      - subject_id
      - name = "MicrobiologyEvents"

19. MICROBIOLOGY EVENT
    Description: Group of microbiology tests from the same specimen
    Properties:
      - event_id
      - hadm_id
      - subject_id
      - micro_specimen_id
      - charttime
      - name = "MicrobiologyEvent"
      - micro_results (array of formatted microbiology result strings)
      - micro_count

20. PROVIDER
    Description: Healthcare provider
    Properties:
      - provider_id (unique identifier)
      - name = "Provider"
      - created_at

21. INITIAL ASSESSMENT
    Description: Triage/initial assessment in ED
    Properties:
      - stay_id
      - name = "InitialAssessment"
      - temperature
      - heartrate
      - resprate
      - o2sat
      - sbp (systolic blood pressure)
      - dbp (diastolic blood pressure)
      - pain
      - acuity
      - chiefcomplaint

22. PATIENT PAST HISTORY
    Description: Patient's historical medical information
    Properties:
      - hadm_id
      - name = "PatientPastHistory"
      - past_medical_history
      - social_history
      - family_history

23. DISCHARGE CLINICAL NOTE
    Description: Clinical notes from discharge
    Properties:
      - note_id
      - hadm_id
      - subject_id
      - name = "DischargeClinicalNote"
      - mental_status
      - level_of_consciousness
      - activity_status
      - discharge_instructions
      - disposition
      - hospital_course
      - imaging_count
      - imaging_studies
      - major_procedure
      - microbiology_findings
      - antibiotic_plan
      - code_status
      - facility_name
      - primary_diagnoses
      - secondary_diagnoses

24. HPI SUMMARY
    Description: History of Present Illness summary
    Properties:
      - hadm_id
      - note_id
      - name = "HPISummary"
      - summary

25. ADMISSION VITALS
    Description: Vital signs at admission
    Properties:
      - vitals_id
      - hadm_id
      - name = "AdmissionVitals"

      - Heart_Rate
      - Blood_Pressure
      - (other dynamic vital properties)

26. ADMISSION LABS
    Description: Lab results at admission
    Properties:
      - labs_id
      - hadm_id
      - name = "AdmissionLabs"

27. ADMISSION MEDICATIONS
    Description: Medications at admission
    Properties:
      - medications_id
      - hadm_id
      - name = "AdmissionMedications"
      - medications (array of medication strings)

28. DISCHARGE VITALS
    Description: Vital signs at discharge
    Properties:

      - note_id
      - name = "DischargeVitals"

29. DISCHARGE LABS
    Description: Lab results at discharge
    Properties:

      - name = "DischargeLabs"

30. DISCHARGE MEDICATIONS

================================================================================
GRAPH PATTERNS AND HIERARCHIES
================================================================================

PATIENT JOURNEY PATTERN:
------------------------
Patient -> HospitalAdmission -> UnitAdmission -> Transfer -> ... -> Discharge
                             |
                             -> EmergencyDepartment (if ED visit)

Each event can have:
  - Prescriptions (PrescriptionsBatch -> Prescription)
  - Lab Events (LabEvents -> LabEvent)
  - Microbiology Events (MicrobiologyEvents -> MicrobiologyEvent)

ICU stays additionally have:
  - Procedures (ProceduresBatch -> Procedures)
  - Chart Events (ChartEventBatch -> ChartEvent)

Hospital admissions additionally have:
  - Procedures (ICD codes)
  - DRG codes
  - Past history
  - HPI summary
  - Admission vitals/labs/medications

Discharges have:
  - Diagnoses (primary and secondary)
  - Discharge clinical notes
  - Discharge vitals/labs/medications
  - Medications started/stopped/to avoid


BATCH-DETAIL PATTERN:
---------------------
Several nodes follow a batch-detail pattern:
  - PrescriptionsBatch -> Prescription (medicines array)
  - ProceduresBatch -> Procedures (procedures array)
  - LabEvents -> LabEvent (lab_results array)
  - MicrobiologyEvents -> MicrobiologyEvent (micro_results array)
  - ChartEventBatch -> ChartEvent (chart_measurements array)


ADMISSION-DISCHARGE PATTERN:
----------------------------
HospitalAdmission
  -> Admission data (vitals, labs, medications)
  -> Events during stay
  -> Discharge
     -> Discharge data (vitals, labs, medications)
     -> Medications changed (started/stopped/avoid)
     -> Diagnoses (primary and secondary)

================================================================================
IMPORTANT NOTES
================================================================================

1. DUAL LABELS: ICUStay nodes have dual labels (:UnitAdmission:ICUStay)

2. CROSS-CONNECTIONS: The graph structure prevents unwanted cross-connections
   between Prescriptions, Procedures, LabEvents, and MicrobiologyEvents

3. ARRAY PROPERTIES: Many nodes store collections as array properties:
   - medicines, procedures, lab_results, micro_results, chart_measurements
   - medications (various medication nodes)
   - diagnoses (complete_diagnosis, primary_diagnoses, secondary_diagnoses)

4. TIME-BASED GROUPING: Events are grouped by time:
   - Prescriptions by starttime
   - Procedures by time
   - Lab events by charttime
   - Microbiology events by specimen_id and charttime
   - Chart events by charttime

5. EVENT-BASED LINKING: Most clinical data (prescriptions, labs, procedures)
   is linked to specific events (UnitAdmission, EmergencyDepartment, ICUStay, Discharge)

6. SEQUENTIAL ADMISSIONS: HospitalAdmissions have sequence numbers, and
   Discharge nodes can link to next HospitalAdmission with gap information


================================================================================
QUERY EXAMPLES
================================================================================

Example 1: Get all admissions for a patient
-------------------------------------------
MATCH (p:Patient {subject_id: 10016742})-[:HAS_ADMISSION]->(ha:HospitalAdmission)
RETURN ha
ORDER BY ha.hospital_admission_sequence_number


Example 2: Get patient journey through hospital
-----------------------------------------------
MATCH path = (p:Patient {subject_id: 10016742})-[:HAS_ADMISSION]->(ha:HospitalAdmission)
              -[:HAS_UNIT_ADMISSION]->(first)
MATCH journey = (first)-[*]->(last)
RETURN path, journey


Example 3: Get all medications during an ICU stay
-------------------------------------------------
MATCH (icu:ICUStay {event_id: "12345"})-[:HAS_PRESCRIPTIONS]->(pb:PrescriptionsBatch)
      -[:HAS_PRESCRIPTION]->(p:Prescription)
RETURN p.medicines, p.starttime
ORDER BY p.starttime


Example 4: Get diagnosis and procedures for admission
-----------------------------------------------------
MATCH (ha:HospitalAdmission {hadm_id: 29079034})
OPTIONAL MATCH (ha)-[:HAS_PROCEDURES]->(pb:ProceduresBatch)-[:HAS_PROCEDURES]->(proc:Procedures)
OPTIONAL MATCH (discharge:Discharge {hadm_id: 29079034})-[:HAS_DIAGNOSES]->(diag:Diagnosis)
RETURN ha, proc.procedures, diag.complete_diagnosis


Example 5: Get lab results during an event
------------------------------------------
MATCH (event {event_id: "12345"})-[:HAS_LAB_EVENTS]->(le:LabEvents)
      -[:HAS_LAB_EVENT]->(lab:LabEvent)
RETURN lab.charttime, lab.lab_results
ORDER BY lab.charttime
